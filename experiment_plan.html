Phase 1
Establish the Problem: Grouping Failure
Do standard defenses collapse when forced to operate on secure groups?

This is the motivating experiment — the entire paper's premise rests on this result. It must be airtight.

  • 4 baselines (FedAvg, Krum, ShieldFL, SignGuard) × 2 modes (Grouped k=10, Non-Grouped)
  • × 2 scenarios (No Attack + Label Flipping) × FEMNIST
  • Fixed: nbyz=20, β=0
  • = 16 runs

Key claim to validate: Krum collapses under grouping (91.7% → 22.4% on MNIST suggests similar on FEMNIST). If ShieldFL or SignGuard also collapse, simplify later comparisons by dropping them.

Backdoor Defense Trade-off
Can FactorGraphs suppress backdoors without destroying model utility?

Switch from Label Flipping to the Scaling Backdoor attack (γ=200, target label 7). Measures both ACC and ASR simultaneously.

  • Reuse FedAvg/Krum grouped results from Phase 1 if applicable

Ideal result: FactorGraphs in the bottom-right quadrant (high ACC, low ASR). FedAvg fails on ASR, Krum fails on ACC.

Phase 4
Non-IID Robustness Sweep
How does FactorGraphs perform across the full spectrum of data heterogeneity?

Sweep all four non-IID levels: β ∈ {0.25, 0.5, 0.75, 1.0}. This validates the CLT-based smoothing claim and maps the degradation curve.

  • FactorGraphs + FedAvg + best surviving baseline from Phase 1
  • × 2 attacks (Label Flipping + Scaling Backdoor)
  • × 4 β values × FEMNIST
  • Fixed: k=10, nbyz=20
  • = 3 methods × 2 attacks × 4 β = 24 runs

Key claim: Distance-based methods (Krum) have high false positive rates under non-IID because benign specialist groups look like outliers. FactorGraphs' temporal tracking resolves this. The full β sweep shows where the crossover happens.

Outcome
Produces a β-sweep line chart (ACC vs β) and adds non-IID columns to all tables. Shows the degradation curve and where FactorGraphs' advantage is largest.

Phase 6
Sensitivity: Group Size × Byzantine Count
How sensitive is FactorGraphs to group size and number of attackers?

Sweep both group size and attacker count to produce a sensitivity heatmap. Preemptively answers a common reviewer question.

  • Group sizes: k ∈ {5, 10, 20, 30}
  • Byzantine clients: nbyz ∈ {10, 20, 30}

Expected trends: Smaller groups → faster detection (less dilution) but more groups needed. More attackers → harder detection but stronger aggregate signal. The heatmap reveals the operating envelope.

Pivot Points
If multiple combos tie in Phase 0
Break ties by practical criteria: training speed, memory footprint, relevance to the target venue (IEEE Trans. Power Systems). Prefer the harder dataset to preempt reviewer pushback.
If Krum doesn't collapse
The "visibility gap" framing weakens. Try smaller group sizes (k=5) or different nbyz before adjusting narrative.
If FactorGraphs underperforms
Stop at Phase 2. Tune belief propagation parameters, observation function, or exclusion threshold before running more.
If β=1.0 breaks everything
Expected — fully non-IID is adversarial to all methods. Report it as an honest limitation rather than omitting it. Show FactorGraphs degrades gracefully.
If MNIST results are trivial
Skip MNIST, invest in CIFAR-10 + ResNet-18 instead. Harder dataset = stronger generalization claim.
If EfficientNet diverges from MobileNetV3
Investigate whether the anomaly scoring function (SignGuard) is architecture-sensitive. May need to tune μmal/μben per architecture.
